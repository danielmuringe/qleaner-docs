# Configuration

Qleaner is configured via a `qleaner.config.json` file in your project root. You can also override many settings using CLI flags.

## Configuration File Structure

The standard configuration file generated by `qleaner init` looks like this:

```json
{
  "codeAlias": "tsconfig.json",
  "paths": {},
  "packageManager": "yarn",
  "excludeDir": ["node_modules", "dist", ".git"],
  "excludeFile": [],
  "excludeExtensions": [".test.tsx", ".spec.ts"],
  "excludeFilePrint": ["index.tsx"],
  "excludeDirAssets": [],
  "excludeFileAssets": [],
  "excludeDirCode": ["node_modules"],
  "excludeFileCode": [],
  "isRootFolderReferenced": true,
  "alias": false
}
```

## Options Reference

| Option | Type | Description |
| :--- | :--- | :--- |
| `codeAlias` | `string` \| `null` | **Critical.** The config file used for path alias resolution (e.g., `"tsconfig.json"`). If set to `null`, `paths` is used manually. |
| `paths` | `object` | Manual alias mapping (e.g., `{"@/*": ["./src/*"]}`). Ignored if `codeAlias` is set. |
| `packageManager` | `string` | Your package manager (`npm`, `yarn`, `pnpm`). |
| `excludeDir` | `string[]` | Directories to ignore during code scans. |
| `excludeFile` | `string[]` | Specific files to ignore during code scans. |
| `excludeExtensions` | `string[]` | File extensions to ignore (e.g., `[".test.js"]`). |
| `excludeFilePrint` | `string[]` | Entry point files. They are scanned but never reported as unused. |
| `excludeDirAssets` | `string[]` | Directories to ignore during image discovery. |
| `isRootFolderReferenced` | `boolean` | Set to `true` if you use absolute paths for images (e.g., `/images/foo.png`). |
| `alias` | `boolean` | Set to `true` if you use alias imports for images. |

## Path Alias Resolution

Accurate path resolution is what allows Qleaner to "connect the dots" between files in complex projects.

*   **Automatic Mode**: When `codeAlias` points to a `tsconfig.json` or `jsconfig.json`, Qleaner automatically reads the `compilerOptions.paths` from that file.
*   **Manual Mode**: If you set `codeAlias` to `null`, you must define the `paths` object in `qleaner.config.json` manually, mimicking the TypeScript path format.

```json
"paths": {
  "@/*": ["./src/*"],
  "components/*": ["./src/components/*"]
}
```

## Default Exclusions

By default, Qleaner ignores common build and system directories to prevent false positives and unnecessary scanning:
*   `node_modules`, `dist`, `build`, `.next`, `out`
*   `coverage`, `.git`, `.cache`
*   Generated folders like `prisma`, `graphql`
